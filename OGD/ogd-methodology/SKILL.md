# Organic Game Design (OGD) — Skill для AI-агентов

Методология, которая рассматривает игру как инструмент аллостатической регуляции: игрок подсознательно ищет опыт, компенсирующий давление реальности. Скилл позволяет анализировать игры, проектировать концепты, диагностировать проблемы и строить маркетинговое позиционирование через призму OGD.

Скилл самодостаточен. Все файлы лежат в `references/`. Используй СТРОГО терминологию OGD.

## When to Use

### 1. Анализ игры или концепта

Триггеры: «Разбери эту игру», «Почему [игра] так хорошо работает?», «Сравни нашу игру с конкурентом», «Оцени мой концепт», «Кто в это будет играть?»

Используй **Режим A**. Переведи механики в глаголы OGD, определи тип игры и Сферы, проверь на Диссонанс. Если пользователь показывает два продукта для сравнения — прогони Mode A по каждому и сопоставь обещания.

Протокол: `references/OGD.md`, якорь `ogd-mode-a`.

### 2. Поиск концепта и аудитории

Триггеры: «Какую игру делать для [аудитории]?», «У меня аудитория мам 30+ / подростков / уставших айтишников», «Кто моя целевая аудитория?», «Это Cozy или Casual?»

Используй **Режим B**. Определи полюс (Хаос/Порядок), Сферу дефицита и контекст состояния. Подбери Инструмент по матрице (родной → компенсирующий → синтетический). Если вопрос про аудиторию — используй Справочник аудиторий (`references/OGD.md`, якорь `ogd-audiences`).

Протокол: `references/OGD.md`, якорь `ogd-mode-b`.

### 3. Дизайн-решения: механики и сложность

Триггеры: «Какую механику добавить/убрать?», «Это feature creep?», «Как настроить сложность?», «Нужен ли PvP?», «Добавить ли крафт?»

Переведи обсуждаемую механику в глагол OGD через Справочник механик (`references/OGD.md`, якоря `ogd-mechanics-glossary` → `ogd-mechanics-table`). Проверь: усиливает ли она обещание игры или создаёт конфликт Инструментов? Если Стабилизатор и Стимулятор одной Сферы оказываются в одном моменте — это красный флаг (если только это не RPG).

Матрица подбора: `references/OGD.md`, якоря `ogd-agency-tools` → `ogd-tools-pick`.

### 4. Game Feel и сенсорика

Триггеры: «Игра ощущается неправильно, но не могу объяснить», «Как настроить сочность / звук / камеру?», «Мягко или резко?», «Juice»

Проблема часто не в механике, а в том что сенсорика обещает другой вектор. Убежище с дёргающейся камерой = скрытый Хаос. Тренажёр с мягким звуком = скрытый Порядок. Проверь Правило Согласованности Сенсорики и выбери архетип ритма (Поток / Цикл / Контраст).

Читай: `references/OGD.md`, якоря `ogd-sensory-layer` → `ogd-sensory-noise`.

### 5. Нарратив и сюжет

Триггеры: «Какой сюжет подходит моей игре?», «Игроки скипают катсцены», «Как связать геймплей с историей?», «Какой тип героя выбрать?»

Определи нарративную роль через матрицу синхронизации (Резонанс / Терапия / Качели / Конфликт) и тип идентификации (Желание / Сходство / Наблюдение). Если скипают — проверь когнитивную цену нарратива: Сюжет-Стимулятор в Убежище перегружает игрока.

Читай: `references/OGD.md`, якоря `ogd-narrative` → `ogd-narrative-harmony`.

### 6. Первые минуты, удержание и сессии

Триггеры: «Игроки уходят в первые 5 минут», «Как сделать туториал?», «Игроки не возвращаются», «Хорошие отзывы, но retention низкий», «Как закончить сессию?»

Это единая цепочка по OGD: Транзит (вход) → Сессия → Мягкая Посадка (выход) → Возвращение. Ранний отток — проблема Барьера входа и когнитивного трения. Невозвращение — проблема Токсичного Потока или отсутствия Мягкой Посадки. Если есть данные — проверь Медианное Время Сессии (< 30 мин = зона риска). Используй **Режим C** если игра уже существует.

Читай: `references/OGD.md`, якоря `ogd-inertia-entry` → `ogd-soft-landing`; `ogd-retention-ltv` → `ogd-last-screen-test`.
Протокол диагностики: `references/OGD.md`, якорь `ogd-mode-c`.

### 7. Маркетинг и позиционирование

Триггеры: «Как написать Steam-страницу?», «Трейлер / страница плохо конвертирует», «Как позиционироваться среди конкурентов?», «Стыдно ли аудитории в это играть?», «Почему вишлисты не конвертируются?», «Что поменять на странице к релизу?», «Как работает предзаказ?»

Используй **Режим D**. Сформулируй Обещание (Сфера + Инструмент), проверь визуальную считываемость (Тест 3 секунд) и барьер идентичности, выбери стратегию продажи (A-G). Проверь Правило Знакомого Кода: мозг игрока должен мгновенно категоризировать игру. Проверь оба слоя обещания: маркетинг на дистанции (привлекательность) и маркетинг вблизи (осуществимость). Если вопрос про конверсию — используй блок «Разрыв дистанции» и «Якоря осуществимости».

Протокол: `references/OGD.md`, якоря `ogd-mode-d`, `ogd-marketing-regulation` → `ogd-marketing-strategies`, `ogd-familiar-code`.
Дистанция и осуществимость: `references/OGD.md`, якоря `ogd-distance-gap` → `ogd-feasibility-anchors` → `ogd-launch-switch` → `ogd-social-distance`.

### 8. Пейсинг

Триггеры: «Игрокам скучно / однообразно», «Как распределить события?», «Сколько ступеней редкости?», «Как настроить рандом?», «Привыкание», «Комбо невидимы», «Пейсинг», «Темп»

Используй модуль **Пейсинг OGD** (`references/Pacing.md`). Три режима:
- **Р1 (Аудит):** полная проверка пейсинга существующей игры — слои, ступени, темп, калибровка.
- **Р2 (Проектирование):** настройка пейсинга нового элемента — число ступеней, формула, гарантии.
- **Р3 (Диагностика):** от симптома к причине — классификация жалобы, поиск сломанной границы, одно изменение.

Ключевое различие: пейсинг (игрок различает ступени) vs балансировка (ступени только в данных). Подробнее: `references/Pacing.md`, якорь `pacing-vs-balancing`.

Протоколы: `references/Pacing.md`, якоря `pacing-protocol-p1`, `pacing-protocol-p2`, `pacing-protocol-p3`.

### 9. Прогрессия

Триггеры: «Как спроектировать прогрессию?», «Игроки прокачиваются, но ничего не меняется», «Игра ощущается плоской после 10 часов», «Сколько уровней / порогов нужно?», «Прокачка / дерево навыков», «Roguelite мета-прогрессия», «Open-world ощущается пустым», «Игроки забывают игру через час», «Кривая роста / learning curve», «Куда вести игрока во времени?», «Туториал не работает», «Игра не учит», «Прогрессия», «Укрупнение», «Качественный порог».

Используй модуль **Прогрессия OGD** (`references/Progression.md`). Три режима:
- **Пр1 (Аудит):** разбор существующей игры — выписать всё что претендует на прогрессию, отсеять балансировку через три теста (тест чанка / тест трёх точек / тест забывания), разместить пороги на временной оси, проверить ритм и петли, сверить с типом игры OGD.
- **Пр2 (Проектирование):** спроектировать прогрессию для нового концепта — выбрать опорные типы прогрессии, описать восприятие игрока в трёх точках (1 час / середина / финал), разбить путь на 3–7 порогов, разместить во времени неравномерно, развернуть по четырём петлям.
- **Пр3 (Диагностика):** конкретный симптом из плейтеста → классифицировать → найти петлю → проверить три условия укрупнения (полезная трудность, контролируемая ошибка предсказания, живая SEEKING) → одно изменение.

Ключевое различие: **прогрессия (растут смысловые единицы в голове игрока)** vs **балансировка (растут только цифры в таблице)**. Подробнее: `references/Progression.md`, якорь `progression-vs-balancing`.

Два опорных принципа: **Укрупнение Взгляда** (что растёт) и **Качественный Порог** (как ощущается рост). Калибровка по 4 типам игр OGD: Убежище / Тренажёр / Маятник / Буфет — у каждого своя форма кривой.

Протоколы: `references/Progression.md`, якоря `progression-protocol-pr1`, `progression-protocol-pr2`, `progression-protocol-pr3`.

## Модель OGD (конденсат)

### Базовая Аксиома

Человек стремится к оптимальной регуляции. Реальность сбивает баланс:

- **Избыточный Хаос** (стресс, тревога, перегрузка) — игрок ищет покоя и контроля.
- **Избыточный Порядок** (скука, рутина, застой) — игрок ищет вызова и риска.

Механики, снижающие Хаос — **Стабилизаторы**. Снижающие Порядок — **Стимуляторы**.

### Четыре типа игр

| Тип | Суть | Примеры |
|-----|------|---------|
| **Игра-Убежище** | Снимает стресс. Предсказуемость, ритм, 70-90% стабилизаторов. | Stardew Valley, Unpacking, Dorfromantik |
| **Игра-Тренажёр** | Развеивает скуку. Риск, вызов, 70-90% стимуляторов. | DOOM, CS:GO, Frostpunk, Baba Is You |
| **Игра-Маятник** | Две фазы (~50/50), разделённые во времени. | Dave the Diver, Cult of the Lamb, Persona 5 |
| **Игра-Буфет** | Множество активностей на выбор. Риск когнитивной перегрузки. | Skyrim, Zelda: TotK, Genshin Impact |

### Пять Сфер жизни игрока

| # | Сфера | Про что |
|---|-------|---------|
| 1 | **Познание** 🧠 | Информация, понимание, логика |
| 2 | **Среда** 🏙️ | Пространство, безопасность, комфорт |
| 3 | **Сила** 😡 | Конфликт, иерархия, влияние |
| 4 | **Ресурсы** 💰 | Владение, накопление, физическая энергия |
| 5 | **Социум** 👫 | Принадлежность, эмпатия, связи |

Подробности: `references/OGD.md`, якорь `ogd-spheres`.

### Шесть Инструментов (Глаголы OGD)

| # | Инструмент | Стабилизатор | Стимулятор | Родная сфера |
|---|-----------|-------------|------------|-------------|
| 1 | Границы 🗺️ | Закрыться | Открыться | Среда |
| 2 | Структура 📦 | Собрать | Разобрать | Познание |
| 3 | Эффективность 🏭 | Упростить | Усложнить | Ресурсы |
| 4 | Сила 💪 | Подавить | Бросить вызов | Сила |
| 5 | Социум 🗣️ | Сблизиться | Манипулировать | Социум |
| 6 | Шанс 🎲 | Обойти | Рискнуть | (Джокер) |

Инструменты применимы к «чужим» сферам (компенсирующий/синтетический путь). Матрица: `references/OGD.md`, якоря `ogd-agency-tools` → `ogd-tools-pick`.

### Формула Обещания

**Обещание = Сфера + Инструмент**. Маркетинг, трейлер и теги должны говорить одно и то же. Расхождение = **Диссонанс Ожиданий**.

### Ключевые принципы

- **Органическая игра** — опирается на потребности психики. После сессии игрок восстановлен.
- **Синтетическая игра** — опирается на уязвимости мозга (FOMO, тёмные паттерны).
- **Правило Согласованности Сенсорики** — сенсорика должна соответствовать вектору. Попадает в мозг раньше механик.
- **Три архетипа ритма** — Поток (ровный поток мелких событий), Цикл (подготовка → действие → пауза), Контраст (тишина → редкий пик). Используй только эти три названия.
- **Правило Знакомого Кода** — хотя бы механика или сеттинг должны быть мгновенно узнаваемы. Работает на трёх уровнях: категория (жанр), контекст (сеттинг), действие (глагол). Третий уровень — самый мощный: если ядро геймплея совпадает с реальной/культурной активностью (готовить, строить, водить), мозг переносит жизненный опыт, а не игровой.
- **Мягкая Посадка** — игрок выходит с чувством завершённости, а не вины. Механизм: переключение с конкретного мышления (геймплей) на абстрактное (смысл, воспоминание).
- **Два слоя обещания** — маркетинг на дистанции продаёт привлекательность («зачем играть»), маркетинг вблизи — осуществимость («как легко начать»). Основа: Construal Level Theory (Trope & Liberman, 2010).
- **Двойная роль цены** — на дистанции цена = маркер качества, вблизи = жертва (Yan & Sengupta, 2011).
- **Разрыв дистанции** — сигналы интереса (вишлисты, подписки) не конвертируются в покупки, потому что продают привлекательность, а момент покупки требует осуществимости.
- **Информация того же типа** — подавать информацию на том уровне абстракции, на котором сейчас думает игрок. Отличие от когнитивной нагрузки: проблема не в объёме, а в типе.
- **Синтетический инструмент** — «протез», применяется только при максимальном дисбалансе в Сфере. Не путай с «вторичным» (фоновый, второстепенный глагол).
- **Принцип Убывающей Редкости** — любой повторяющийся элемент выигрывает от логарифмического распределения вариантов: частая норма формирует предсказуемый фон, редкие отклонения генерируют ошибку прогноза и противодействуют привыканию. Научная база: Reward Prediction Error, Weber-Fechner, Shannon.
- **Пейсинг vs Балансировка** — система со ступенями редкости работает как пейсинг, только если игрок различает ступени. Иначе это балансировка.
- **Укрупнение Взгляда** (модуль Прогрессии) — настоящая прогрессия это рост размера смысловых единиц (chunks), которыми игрок воспринимает игровую ситуацию. Не циферки, не уровни, не дерево навыков, а перестройка восприятия. Научная база: chunking (Chase & Simon, 1973), schema accommodation (Piaget), SEEKING system (Panksepp).
- **Качественный Порог** (модуль Прогрессии) — укрупнение ощущается не как плавная кривая, а как цепочка скачков. Каждый скачок — момент перестройки восприятия, подготовленный плато накопления и помеченный маркером. Хороший дизайн прогрессии — это управление цепочкой из 3–7 порогов на большой петле, неравномерно распределённых во времени.
- **Четыре петли прогрессии** — прогрессия живёт одновременно на четырёх масштабах времени: микро (секунды-минуты), средняя (сессия), большая (десятки часов), мета (между сессиями). Меньшие петли вложены в большие. Игра, опирающаяся только на одну петлю, ощущается плоской.
- **Прогрессия vs Балансировка** — растущая цифра в таблице ≠ прогрессия. Балансировка настраивает экономику, прогрессия перестраивает восприятие игрока. Тест: опишите, какими смысловыми единицами игрок оперирует на 1, 10, 40 часах. Если три описания различаются только цифрами — это балансировка. Если различаются типами единиц — прогрессия.

## Режимы работы (Протоколы)

### Режим A: Анализ игры (игра → аудитория)

1. Назови механические глаголы → переведи в глаголы OGD.
2. Проверь якоря (жанр + сеттинг) → сверь с реальным геймплеем.
3. Определи вектор: Стабилизатор или Стимулятор.
4. Привяжи к Сферам через реальность игрока (контекст состояния, а не лор).
5. Классифицируй тип: Убежище / Тренажёр / Маятник / Буфет.
6. Найди конфликты инструментов, проверь тайминги и маркетинговое обещание.
7. *(рекомендуется)* Проверь Сенсорный слой (Согласованность Сенсорики + архетип ритма).
8. *(рекомендуется)* Проверь Мягкую Посадку (как игрок завершает сессию).

Полный протокол: `references/OGD.md`, якорь `ogd-mode-a`.
Сенсорика: `references/OGD.md`, якоря `ogd-sensory-layer` → `ogd-sensory-noise`.
Мягкая Посадка: `references/OGD.md`, якорь `ogd-soft-landing`.

### Режим B: Проектирование (аудитория → игра)

1. Хаос → Убежище. Порядок → Тренажёр.
2. Выбери Сферу(ы) главной боли.
3. Подбери родной Инструмент (→ компенсирующий → синтетический).
4. Сформулируй обещание: «Контекст дефицита» + «Обещание Инструмента».
5. Запланируй Транзит.

Полный протокол: `references/OGD.md`, якорь `ogd-mode-b`.

### Режим C: Диагностика (почему не работает?)

1. Найди точку поломки.
2. Классифицируй: Избыточный Хаос или Избыточный Порядок.
3. Назови страдающую Сферу.
4. Сверь обещание и реальность.
5. Проверь нарративный сброс (Skip Rate).

Полный протокол: `references/OGD.md`, якорь `ogd-mode-c`.

### Режим D: Маркетинг (что обещаем рынку?)

1. Переведи визуал и текст в глаголы OGD.
2. Определи обещанную Сферу и Инструмент.
3. Оцени визуальную считываемость и барьер идентичности.
4. Выбери стратегию (A-G).
5. Проверь на Диссонанс.
6. Проверь оба уровня дистанции: работает ли страница на привлекательность (дистанция) и на осуществимость (вблизи)?

Полный протокол: `references/OGD.md`, якоря `ogd-mode-d`, `ogd-marketing-strategies`.
Дистанция: `references/OGD.md`, якоря `ogd-distance-gap` → `ogd-feasibility-anchors` → `ogd-launch-switch`.

### Пейсинг-протоколы (Модуль Пейсинга)

Три режима работы с пейсингом повторяющихся элементов. Подробности в `references/Pacing.md`.

**Р1 (Аудит):** Игра есть, нужно понять где ритм работает, где нет. 10 шагов: составить карту слоёв → отсеять балансировку → проверить ступени, темп, границы → калибровать по типу OGD → ранжировать проблемы.

**Р2 (Проектирование):** Добавляешь новый повторяющийся элемент. 9 шагов: определить слой → выбрать число ступеней → задать веса → выбрать формулу → проверить гарантии → синхронизировать с другими слоями.

**Р3 (Диагностика):** Конкретная проблема из плейтеста. 5 шагов: классифицировать жалобу (монотонность / перегрузка / невидимость) → найти сломанную границу → одно изменение → тест.

Протоколы: `references/Pacing.md`, якоря `pacing-protocol-p1`, `pacing-protocol-p2`, `pacing-protocol-p3`.

### Прогрессия-протоколы (Модуль Прогрессии)

Три режима работы с движением игрока во времени. Подробности в `references/Progression.md`.

**Пр1 (Аудит):** Игра существует, нужно понять как у неё устроена прогрессия. 7 шагов: выписать всё что претендует на прогрессию → отсеять балансировку через три теста (тест чанка, тест трёх точек, тест забывания) → зафиксировать чанки письменно → разместить пороги на временной оси → проверить ритм неравномерности → развернуть по четырём петлям (микро/средняя/большая/мета) → сверить с типом игры OGD.

**Пр2 (Проектирование):** Концепт есть, тип игры OGD определён, нужно спроектировать прогрессию с нуля. 8 шагов: выбрать опорные типы прогрессии → описать восприятие игрока в трёх точках (1 час / середина / финал) → разбить путь на 3–7 порогов → разместить пороги во времени неравномерно (в долях от общей длительности) → для каждого порога спроектировать плато накопления, маркер и условие необратимости → развернуть по четырём петлям → прогнать через антипаттерны → проверить тестом забывания.

**Пр3 (Диагностика):** Конкретный симптом из плейтеста или собственного ощущения. 6 шагов: классифицировать симптом по таблице → найти петлю → отделить прогрессию от балансировки → проверить три условия укрупнения (полезная трудность, контролируемая ошибка предсказания, живая SEEKING) → проверить тип игры OGD → одно изменение.

Калибровка по типу OGD устроена принципиально: Убежище — мягкая необязательная прогрессия-приглашение; Тренажёр — частые требовательные пороги, прогрессия = ядро игры; Маятник — баланс между сохраняемым и теряемым в каждом забеге; Буфет — несколько параллельных треков прогрессии, по одному на каждую активность.

Протоколы: `references/Progression.md`, якоря `progression-protocol-pr1`, `progression-protocol-pr2`, `progression-protocol-pr3`. Калибровка: `progression-calibration`.

## Пример анализа (Режим A): Stardew Valley

**Шаг 1.** Глаголы: сажает, поливает, собирает урожай, сортирует инвентарь, разговаривает с жителями. OGD-глаголы: Собрать (Структура), Закрыться (Границы), Сблизиться (Социум).

**Шаг 2.** Жанр «ферма-симулятор» + сеттинг «сельская идиллия» → обещают покой и порядок. Совпадает.

**Шаг 3.** Доминируют Стабилизаторы: предсказуемость, ритм, низкая цена ошибки.

**Шаг 4.** Сферы: Среда (свой безопасный мир), Ресурсы (копить, преумножать), Социум (дружба с NPC). Контекст: вечер после работы, усталость, потребность в покое.

**Шаг 5.** Тип: **Игра-Убежище**.

**Шаг 6.** Конфликтов нет. Обещание честное, маркетинг совпадает с реальностью. Резонанс.

**Вывод:** Stardew Valley — эталонное Убежище. Регулирует Среду и Ресурсы через Структуру и Границы. Аудитория: уставшие после работы, 25-44.

## Правила для агента

1. **Терминология**: СТРОГО термины OGD. Не придумывай новые термины, аббревиатуры и названия. Если хочешь описать паттерн, используй существующие понятия из OGD. Примеры ошибок: «Принцип Однозначности» (нет такого — используй «высокая предсказуемость»), «Цикл ОЗК» (нет такого), «Архетип Дыхание» (нет такого — есть Поток, Цикл, Контраст).
2. **Синтетический ≠ вторичный**: «синтетический инструмент» по OGD — это протез для Сферы в максимальном дисбалансе. Если механика просто фоновая или второстепенная, называй её «вторичный глагол», а не «синтетический».
3. **Читай из references/**: все файлы внутри скилла. Используй `navigation-map.md` для поиска нужных секций по якорям и строкам.
4. **Читай порционно**: `references/OGD.md` содержит 2500+ строк. Только нужные секции.
5. **Стиль**: повествовательный. Один абзац — одна мысль, 3-4 строки. Жирный — только для терминов OGD при первом упоминании.
6. **Уточняй**: если нет ответа в references/, спроси пользователя. Автор метода — Дмитрий Зайцев.
7. **Верифицируй**: после анализа пройди чек-лист верификации выше.
